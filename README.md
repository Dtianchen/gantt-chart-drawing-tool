# 📊 甘特图画图工具

<p align="center">
  <strong>一款轻量级的项目进度管理工具</strong><br>
  基于 React + TypeScript + Vite 构建，支持 Web 浏览器和 Windows 桌面端
</p>

---

## ✨ 功能特性

| 功能 | 说明 |
|------|------|
| 📅 **甘特图展示** | 时间轴视图直观显示任务进度条 |
| ✏️ **任务管理** | 添加、编辑、删除、拖拽排序任务 |
| 🔄 **时间调整** | 通过拖拽修改任务的开始/结束日期 |
| 📏 **工程标尺** | 每5天显示刻度数字（0, 5, 10...） |
| ⏱️ **工期统计** | 自动计算并显示项目总工期 |
| 🔴 **今日线** | 在时间轴上标记当前日期位置 |
| 🔍 **缩放控制** | 支持日/周/月三种时间粒度切换 |
| 🟨 **周末高亮** | 自动标识周六/周日（红色背景） |
| 📸 **导出图片** | 将甘特图导出为 PNG 图片 |
| 💾 **数据持久化** | 使用 localStorage 自动保存任务数据 |
| 🖥️ **桌面应用** | 支持 Electron 打包为 EXE 可执行文件 |

## 🚀 快速开始

### 环境要求

- **Node.js** >= 18.0.0
- **npm** >= 9.0.0

> 💡 **没有 Node.js 环境？** 使用 [便携版 ZIP](#便携版)，解压即用！

### 安装与运行

```bash
# 克隆仓库
git clone https://gitee.com/tianchen007/gantt-chart-drawing-tool.git
cd gantt-chart-drawing-tool

# 安装依赖
npm install

# 启动开发服务器
npm run dev
# 或双击 start.bat
```

浏览器访问 http://localhost:5173 即可使用。

### 一键脚本

| 脚本 | 用途 |
|------|------|
| `start.bat` | 开发模式（自动检测并安装依赖） |
| `build.bat` | 构建生产版本（自动检测并安装依赖） |
| `启动进度工具.bat` | 便携版启动（使用内置 Node.js） |
| `构建生产版本.bat` | 便携版构建（使用内置 Node.js） |
| `打包EXE.bat` | 打包 Windows 桌面应用 |

## 📖 使用说明

### 基本操作

1. **添加任务**：点击工具栏「+ 添加任务」按钮
2. **编辑任务**：点击任务名称进入编辑模式
3. **删除任务**：点击任务行右侧删除图标
4. **调整时间**：拖拽任务条的左/右边缘调整开始/结束日期
5. **排序任务**：拖拽任务行左侧 ⋮ 图标调整顺序
6. **导出图片**：点击工具栏「导出图片」按钮下载 PNG

### 快捷操作

- 点击 **今日** 按钮快速定位到当前日期
- 使用 **缩放控件** 切换日/周/月视图
- 所有修改自动保存到浏览器本地存储

## ❓ 常见问题

### Q: 每次打开都需要重新安装依赖吗？

**A:** 这是正常现象！`node_modules/` 目录被 `.gitignore` 忽略（不提交到 Git），因为它的体积很大（~100MB+）。每次克隆或拉取代码后需运行 `npm install`。也可以使用一键脚本 `start.bat`，它会自动检测并安装缺失的依赖。

### Q: 导出的图片不完整怎么办？

**A:** 请确保：
1. 浏览器已滚动到甘特图最顶部和最左侧
2. 尝试缩小缩放比例后再导出
3. 使用 Chrome/Edge 浏览器获得最佳效果

### Q: 数据保存在哪里？

**A:** 任务数据存储在浏览器的 localStorage 中，清除浏览器数据会导致丢失。建议定期使用「导出图片」备份。

### Q: 如何在没有 Node.js 的电脑上使用？

**A:** 使用便携版 ZIP 包，已内置完整的 Node.js 运行时环境，解压后双击即可运行。
